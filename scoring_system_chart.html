Pedigree JSON Scoring System

Comprehensive evaluation framework for pedigree accuracy assessment

Starting Point

100 POINTS

Deductions are applied based on discrepancies between golden and test JSONs

TIER 1: FOUNDATION (50% Weight)

Generation Structure -5 to -15 pts

• 1 generation difference: -5 points
• 2+ generation differences: -15 points

Nodes per Level Distribution -2 pts × weight

• Generation-weighted penalties
• Higher generations get more weight
• Exponential decay: 1.0/(1.2^level)

Core People Detection -5 pts each

• Male/Female count differences
• Each discrepancy penalized

Miscarriage Detection -1 pt each

• Reduced weight for miscarriage nodes
• Less critical than core people

TIER 2: RELATIONSHIPS (40% Weight)

Parent Relationships -4 pts each

• Nodes marked as having no parents (no_parents flag)
• Nodes that should/shouldn't have parents
• Accuracy of parent-child connections

Partner Relationships -4 pts each

• Nodes with exactly one partner
• Nodes with multiple partners

Sibling Relationships -4 pts each

• Total nodes with siblings
• Distribution of siblings (1, 2, 3, 4, 5+ siblings)

TIER 3: ATTRIBUTES (10% Weight)

Disease Patterns -1 pt each

• Medical condition symbols
• Shading pattern differences
• Disease inheritance markers

Important Symbols -2 pts each

• Deceased status
• Adopted_in/Adopted_out
• Critical family markers

Twin Relationships -1 pt each

• DZ twin (dizygotic) differences
• MZ twin (monozygotic) differences

Score Calculation Formula

total_deductions = (tier1_deductions × 0.50) + (tier2_deductions × 0.40) + (tier3_deductions × 0.10) final_score = max(0.0, 100 - total_deductions)
